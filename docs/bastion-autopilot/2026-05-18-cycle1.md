## 발견 결함 3 (severity 순)

### 결함 1 (Critical) — LLM Hallucination
- skill_result success=false 인데 _synth_prompt 가 LLM 에게 "도구 stdout 에서
  인용" 요구 → empty output 시 모델 (gemma3:4b) 이 가짜 example 생성
- 학습 가치 0 + 운영 위험 (가짜 보고서)

**Root cause**: `packages/bastion/agent.py:2360-2368` 의 `_synth_prompt` 가
"실측 기반" 강제 했지만 empty stdout 분기 없음. 작은 모델 (gemma3:4b) 의
instruction following 약함.

### 결함 2 (High) — Skill target 잘못
- LLM 이 docker ps 의 target 을 `10.20.30.1` (fw) 으로 선택
- 정답 = bastion 자신 (docker.sock RO mount 가용)
- shell skill 의 `target` 파라미터 description 약함 ("대상 VM role")

### 결함 3 (High) — KG anchor 의 false-positive 기록
- `kg_status.record success=true` — 가짜 응답 이 task_outcome anchor 로 기록
- 다음 lookup 시 sim 매치 시 같은 가짜 응답 재사용 위험 (PE-KG 학습 의 데이터
  오염)
